Introduction

Notification modules allow the SQL Relay server programs to notify recipients when a specified set of events occur. A section of the sqlrelay.conf file indicates which modules to load and what parameters to use.

For example, to use the events module, you would include a section in the sqlrelay.conf file like:

<?xml version="1.0"?>
<instance ...>
        ...
        <notifications>
                <notification module="events">
                        <events>
                                <event event="db_login"/>
                                <event event="db_logout"/>
                                <event event="filter_violation"/>
                        </events>
                        <recipients>
                                <recipient address="david.muse@firstworks.com"/>
                        </recipients>
                </notification>
        </notifications>
        ...
</intstance>

The module attribute specifies which module to load. The child tags and attributes indicate which events to listen for and which recipients to send notifications to.

Different modules may have different parameters.

Currently, all notification modules have an enabled parameter, allowing the module to be temporarily disabled. If enabled="no" is configured, then the module is disabled. If set to any other value, or omitted, then the module is enabled.

Multiple notification modules may be loaded by specifying multiple notification tags.

Internally, at startup the SQL Relay server processes create instances of the specified notification modules and initialize them. Then, as they run, the call the notification modules' "run" method, passing in an event type, and information about the event. The modules then react to the different events, in a module-specific manner.
