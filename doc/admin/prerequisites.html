Platforms

SQL Relay is supported on a variety of Linux/Unix systems and Windows. Building SQL Relay on any of these platforms requires the installation of a compiler toolchain and other bits of optional software. Instructions follow for many supported platforms.

Linux

SQL Relay is supported on a variety of Linux platforms including Fedora, CentOS, OpenSuSE, Debian, and Ubuntu.

Fedora

To install all prerequisites at once, execute the following command, as root, or using sudo:

dnf -y install gcc-c++ readline-devel perl-devel php-devel php-pdo python-devel ruby-devel ruby-libs tcl-devel java-1.8.0-openjdk-devel erlang nodejs-devel node-gyp mono-devel mono-data-oracle mariadb-devel postgresql-devel firebird-devel firebird-libfbclient sqlite-devel unixODBC-devel freetds-devel mdbtools-devel mdbtools-libs

Note that modern Fedora platforms use the dnf package manager but older Fedora platforms use the yum package manager. The syntax for the two is the same. For older Fedora platforms, just replace dnf with yum in the command above.

Note also, that on older Fedora platforms, JDK 1.8.0 is not available so java-1.8.0-openjdk-devel must be replaced with java-1.7.0-openjdk-devel.

Client API's for Oracle, SAP/Sybase, DB2, and Informix are not available from dnf/yum repositories and must be installed separately.

The simplest way to install the Oracle client API is to download the Oracle Instant Client RPM's from Oracle and install them using the following commands, as root:

dnf install libaio
rpm -i oracle-instantclient12.1-basic oracle-instantclient12.1-devel

For SAP/Sybase, you have to download the ASE installer from SAP and install the client components. See Getting Started with SAP/Sybase for more information.

For DB2, you have to download the installer from IBM and perform a full installation. See Getting Started with IBM DB2 for more information.

For Informix, you also have to download the installer from IBM and perform a full installation, though it isn't necessary to create a local instance.
